# 🌟 Эхо Света

**«Эхо Света»** — это медитативный 2D-симулятор выживания в условиях угасающей экосистемы. Вы играете за Искру — последний активный импульс заброшенной нейросети (или угасающего леса), для которого движение — это единственный способ вдохнуть жизнь в умирающие узлы данных. Ваша цель: восстановить разрушенную сеть и удержать её стабильность под натиском энтропии.

Действие разворачивается на темной, лаконичной арене. Минималистичный визуальный стиль (контраст глубокого черного фона и яркого неонового свечения) выбран специально: здесь нет визуального шума, что позволяет полностью сконцентрироваться на тактическом планировании маршрута и удержании баланса системы в условиях жесткого дефицита времени.

---

### 🎯 Ключевые механики

Главная особенность игры — поддержание стабильности динамической системы, где каждый узел постоянно теряет энергию, а связи между ними изначально разорваны.

*   **Зарядка:** Нахождение Искры (игрока) вблизи узла постепенно заполняет его шкалу энергии. 
*   **Ремонт связей:** Некоторые каналы связи между узлами повреждены. Игрок должен физически встать на поврежденную линию и удерживать позицию в течение 2 секунд, чтобы восстановить ребро графа.
*   **Цепная реакция (BFS):** Когда узел заряжается до 100%, он автоматически активирует алгоритм обхода графа в ширину (BFS) и передает излишки своей энергии соседним узлам по активным (отремонтированным) связям.
*   **Удержание (Hold-to-Win):** Победа не дается мгновенно. Когда средняя энергия всей сети превышает 85%, запускается таймер финальной синхронизации. Игроку необходимо удержать этот порог стабильности в течение 3 секунд.

### 👾 Бестиарий и элементы сети

На игровом поле генерируются интерактивные объекты двух типов:

*   **Узлы связи (Золотые / Серые):** Вершины графа. В зависимости от текущего заряда меняют свой цвет (от безжизненного серого до неонового золотого). Обладают процедурной анимацией «дыхания».
*   **Теневые пиявки (Фиолетовые):** Автономные противники. Они хаотично перемещаются по арене, но при сближении с узлом переходят в режим преследования. Достигнув узла, пиявка начинает стремительно высасывать из него энергию. Игрок должен уничтожить пиявку, протаранив её своей Искрой.

### 🎮 Управление и Интерфейс

Вся навигация и игровой процесс контролируются исключительно с клавиатуры:

*   **W, A, S, D / Стрелки** — Плавное движение Искры / Навигация по пунктам меню.
*   **ПРОБЕЛ** — Быстрый пропуск интерактивного обучения.
*   **ENTER** — Перезапуск игры после победы или поражения / Возврат в меню.

Интерфейс спроектирован по принципу максимальной информативности:
*   Вокруг игрока всегда виден бирюзовый ореол, наглядно показывающий радиус его взаимодействия.
*   Над поврежденными связями во время ремонта плавно заполняется локальный прогресс-бар.
*   Внизу экрана расположен неоновый HUD со шкалой общей стабильности сада (среднее арифметическое энергии всех узлов), на которой при достижении 85% визуализируется бирюзовый таймер удержания победы.

### 💻 Техническая реализация (Под капотом)

Игра представляет собой стабильный, оптимизированный MVP, написанный с нуля на языке C# с использованием библиотеки GDI+ фреймворка Windows Forms.

Архитектура строго соответствует академическим стандартам и принципам чистого кода:

*   **Архитектурный паттерн MVC:** Проект разделен на три независимые папки:
    *   `Model` — хранит только структуры данных (`Flower`, `Player`, `Leech`) и чистую математику. Она полностью изолирована от графического фреймворка.
    *   `View` — содержит класс `GameRenderer`, который отвечает исключительно за отрисовку графики на основе данных модели.
    *   `Controller` — класс `MainForm`, связывающий пользовательский ввод, таймер и запуск перерисовки.
*   **Паттерн «Наблюдатель» (Observer):** Реализован встроенными средствами C# через события (`event Action OnStateChanged`). Отрисовка не опрашивает модель вхолостую — форма перерисовывается (`Invalidate()`) только тогда, когда модель сообщает о завершении шага расчета физики.
*   **Высокоточный игровой цикл (DeltaTime):** Вместо неточного системного таймера расчет физики завязан на высокоточный секундомер `Stopwatch`. Движения объектов, угасание энергии и плавные переходы (`Lerp`) рассчитываются относительно реального времени кадра (`dt`), что гарантирует плавность 60 FPS на любом мониторе.
*   **Процедурная графика:** Все визуальные эффекты (свечение, пульсация, плавные переходы цветов) рассчитываются математически в реальном времени. Пульсация Искры завязана на синусоиду времени `Math.Sin(Time)`.
*   **Графовые алгоритмы:** Связи между узлами представляют собой структуру неориентированного графа. Распространение энергии реализовано через классический алгоритм BFS (Breadth-First Search) с оценкой сложности $O(V+E)$, что гарантирует мгновенную работу без фризов.
*   **Плавная физика:** Движение Искры основано на векторной физике с учетом сил ускорения, инерции и силы трения, что обеспечивает комфортное и отзывчивое управление.

### 🚀 Как запустить проект

Для запуска игры не требуются сторонние движки вроде Unity.

1. Убедитесь, что у вас установлена среда разработки **Visual Studio** (с поддержкой рабочей нагрузки *.NET Desktop Development*).
2. Скачайте репозиторий с проектом из вашего репозитория GitHub или распакуйте архив в удобную папку.
3. Откройте файл решения **`LightEcho.sln`** в Visual Studio.
4. Если при импорте решения возникла ошибка совместимости версий, просто откройте файл проекта **`LightEcho.csproj`** напрямую через проводник.
5. Нажмите клавишу **`F5`** или кнопку **Пуск (Start)** на верхней панели для автоматической сборки и запуска игры.
